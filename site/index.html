Repo Contracts AI-Human Teams Auto Cleanup

Repotype is the auto-vacuum for AI-generated repo entropy.

Claude, ChatGPT, Copilot, and local agents can generate useful output fast, but they can also scatter files, invent folders, and drift your structure. Repotype adds friction in the right places so your repository stays clean, predictable, and maintainable without constant manual cleanup.

Why this matters now

AI can write faster than teams can review. Entropy wins unless structure is enforced by default.

In most teams, the first phase of AI adoption increases output and decreases consistency. You get extra markdown, duplicate requirement files, scripts parked in random folders, and subtle naming drift. None of this is dramatic in one PR, but after a few weeks it creates drag everywhere: slower onboarding, brittle automation, and arguments about where content should live.

Repotype solves that by defining your repo as a contract. It decides what belongs where, what metadata is required, what sections are mandatory, and how cross-references must resolve. If a file does not fit, it fails clearly or gets queued for cleanup with logs. The result: less policing, less cleanup labor, and more trust in agent output.

Agent Sprawl Control

Stop random docs and code from being dropped in arbitrary places.

  • Deny-by-default unmatched files
  • Folder allowlists and required folders
  • Path naming and case policies

Document Quality Guardrails

Require frontmatter + structure so docs remain usable for automation.

  • JSON Schema validation
  • Required markdown sections
  • Template compliance checks

Automatic Hygiene

Use hooks/watchers to keep the repo tidy continuously, not occasionally.

  • Pre-commit/pre-push enforcement
  • Cleanup queue + move logs
  • Machine-readable reports for CI

Quickstart

pnpm --filter repotype build
pnpm --filter repotype exec repotype init .
pnpm --filter repotype exec repotype validate . --json
pnpm --filter repotype exec repotype apply .

Operational Signals

Repotype outputs clear severity so teams and agents know what to do next.

Error: contract violation Warning: policy risk Pass: repo conforms
