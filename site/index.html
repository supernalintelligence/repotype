Repo Contracts Agent-Safe CI-Enforced

Stop agent-generated repo sprawl before it lands in main.

Repotype defines and enforces how your repository should be structured: folders, file patterns, markdown/frontmatter rules, naming conventions, companion files, and cleanup policy.

View on GitHub Read docs

Agent Output Control

Prevent “docs/code anywhere” drift by requiring explicit allow rules for where generated content can live.

  • Deny-by-default unmatched files
  • Folder allowlists and required paths
  • Path case + naming enforcement

Doc Contract Enforcement

Ensure markdown is consistently structured and machine-usable for planning, testing, and traceability.

  • Required frontmatter schema
  • Required sections per doc type
  • Template enforcement + cleanup hints

Operational Hygiene

Install hooks and watcher policies from config to catch drift continuously, not just at review time.

  • Pre-commit / pre-push checks
  • Cleanup queue with audit logs
  • JSON report output for automation

Workflow

  1. Define repository contract in repotype.yaml.
  2. Run repotype validate . locally and in CI.
  3. Use repotype fix and cleanup-run for safe remediation.
  4. Publish evidence with repotype report --json.

Quickstart

pnpm --filter @supernal/repotype build
pnpm --filter @supernal/repotype exec repotype init .
pnpm --filter @supernal/repotype exec repotype validate . --json
pnpm --filter @supernal/repotype exec repotype apply .
Error: repo contract violation Warning: weak policy or mismatch risk Pass: repo typed + enforceable
